    Props interface:
    ```typescript
    interface MainViewProps {
      token: string;
    }
    ```

    State:
    - `urlInput`: string (raw URL text)
    - `parsedUrl`: FigmaUrlParts | null (parsed result)
    - `scope`: ExtractionScope ('node' | 'frame' | 'page')
    - `fileInfo`: FigmaFileInfo | null (validated file metadata)
    - `validating`: boolean (API call in progress)
    - `error`: string | null (error message)

    **URL Input Section (className="figma-plugin-section"):**
    - Label "Figma URL" (className="figma-plugin-label")
    - Text input (className="figma-plugin-input"), placeholder="https://www.figma.com/design/..."
    - On change/paste: update urlInput, call parseFigmaUrl(value)
    - If parsed result is non-null: update parsedUrl, auto-set scope default, trigger file validation
    - If parsed result is null and input is not empty: show "Invalid Figma URL" error
    - If input is empty: clear everything (parsedUrl, fileInfo, error)

    **Auto-scope defaulting (when parsedUrl changes):**
    - If parsedUrl.nodeId is not null: default scope to 'node'
    - If parsedUrl.nodeId is null: default scope to 'page'

    **Parsed URL info display (when parsedUrl is non-null):**
    - Show parsed info in a subtle card (className="figma-plugin-file-info"):
      - File key: `{parsedUrl.fileKey}` (truncated if very long)
      - Node: `{parsedUrl.nodeId || 'None (file-level)'}`
      - Type: `{parsedUrl.fileType}`
    - If validating: show spinner (className="figma-plugin-spinner") with "Checking access..."
    - If fileInfo loaded: show file name and page count in the info card

    **File validation (triggered automatically when parsedUrl changes):**
    - Call `validateFileAccess(shell, token, parsedUrl.fileKey)`
    - On success: set fileInfo (shows file name, page list)
    - On failure: set error message (could be 403 "no access", 404 "not found", etc.)
    - Use a debounce or simply trigger on parsedUrl change (since parseFigmaUrl is called on every input change, only validate when a valid parse result appears)
    - IMPORTANT: If user changes the URL while a validation is in flight, abort the stale result. Use a simple counter/flag approach: increment a request ID before each call, check it hasn't changed when the response arrives.

    **Scope Selection Section (className="figma-plugin-section", shown when parsedUrl is non-null):**
    - Label "Extraction Scope" (className="figma-plugin-label")
    - Radio group (className="figma-plugin-radio-group") with three options:
      1. "Single Node" (value='node') -- only enabled when parsedUrl.nodeId is not null. If nodeId is null, disable this option and show hint "Paste a URL with a node-id to use this option"
      2. "Frame" (value='frame')
      3. "Entire Page" (value='page')
    - Each radio: `<label className="figma-plugin-radio-label"><input type="radio" .../> Label text</label>`

    **Extract Button (bottom of view):**
    - `<button className="btn-primary">` with text "Extract Design Brief"
    - Disabled when: no parsedUrl, no fileInfo (validation not complete), or validating
    - For Phase 1: clicking the button shows a toast "Extraction coming in next update" via actions.showToast. The actual extraction logic will be wired in Phase 2.
    - Full width (style={{ width: '100%' }})

    **Error handling:**
    - Display errors below the URL input with className="figma-plugin-error"
    - Common error messages:
      - Invalid URL: "Please paste a valid Figma URL (file, design, proto, or board link)"
      - No access (403): "Cannot access this file. Check that your token has File content (Read) scope."
      - Not found (404): "File not found. Check that the URL is correct."
      - Rate limited (429): "Rate limited by Figma. Please wait a moment and try again."
