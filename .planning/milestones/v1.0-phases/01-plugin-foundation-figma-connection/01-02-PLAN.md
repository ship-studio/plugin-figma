    **Step 2: Update src/index.tsx to wire view routing**

    Replace the placeholder modal body with conditional view rendering.

    The FigmaToolbarButton component manages:
    - `modalOpen`: boolean (modal visibility)
    - `token`: string | null (stored Figma PAT)
    - `figmaUser`: FigmaUser | null (validated user info)
    - `loaded`: boolean (storage read complete)
    - `currentView`: 'main' | 'settings' (which view to show)

    **On mount (useEffect):**
    - Call `storage.read()` to check for existing token
    - If `data.figmaToken` is a string, set token and figmaUser (from data.figmaUserHandle or re-validate)
    - Set loaded=true
    - Guard renders with: if (!loaded) return toolbar button only (no modal content until storage read)

    **View routing inside Modal:**
    - If no token → render `<SetupView onTokenSaved={handleTokenSaved} />`
    - If token exists and currentView === 'settings' → render `<SettingsView ... />`
    - If token exists and currentView === 'main' → render placeholder "Main view coming in Plan 03" (temporary)

    **Handlers:**
    - `handleTokenSaved(token, user)`: Save to storage.write({ figmaToken: token, figmaUserHandle: user.handle }), update state, show success toast via actions.showToast
    - `handleTokenUpdated(token, user)`: Same as saved, switch back to main view
    - `handleTokenRemoved()`: Read storage, delete figmaToken and figmaUserHandle keys, write back, set token=null, figmaUser=null, currentView='main'

    **Modal header:**
    - Title: "Figma" (or "Figma Design Brief")
    - headerRight: When token exists, render a gear icon button to switch to settings view. Use a simple SVG gear icon (14x14). On click, set currentView='settings'.

    **IMPORTANT:** All storage operations are async. Use try/catch around storage.write() and storage.read(). If storage fails, show error via actions.showToast.
