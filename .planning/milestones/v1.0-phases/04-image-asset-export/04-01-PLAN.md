    - Node with IMAGE fill but NOT in ImageFillRef list -> still identified (imageRef from fill paint directly)

    - Hidden nodes (visible: false) -> still exported (user may want hidden assets)

    **Step 5: Implement identifyAssets**

    In `src/assets/identify.ts`:
    ```typescript
    import type { LayoutNode } from '../layout/types';
    import type { ImageFillRef } from '../tokens/types';
    import type { AssetEntry } from './types';
    import { sanitizeFilename, resolveCollisions } from './sanitize';

    const SVG_TYPES = new Set(['VECTOR', 'BOOLEAN_OPERATION', 'LINE', 'STAR', 'POLYGON', 'ELLIPSE']);

    export function identifyAssets(rootNodes: LayoutNode[], imageFills: ImageFillRef[]): AssetEntry[]
    ```

    Walk logic:
    1. Build a Set of nodeIds from imageFills for quick lookup, and a Map from nodeId to imageRef.
    2. For each rootNode, scan its direct children (top-level). For each child:
       a. If `child.type === 'INSTANCE'` -> add as SVG entry. Do NOT recurse into INSTANCE children.
       b. If child has fills with `type === 'IMAGE'` -> add as `png-fill`. Use imageRef from the imageFills Map if available, else extract from `child.fills.find(f => f.type === 'IMAGE').imageRef`.
       c. If `child.type === 'RECTANGLE'` -> check fills first. If has IMAGE fill, treat as png-fill (rule above). Otherwise treat as SVG.
       d. If `SVG_TYPES.has(child.type)` -> add as SVG entry.
       e. If child is a FRAME/GROUP/SECTION (container), recurse ONE level into its children applying same rules (component-level). But do NOT recurse further -- this gives "top-level and component-level only."
    3. For imageFills that didn't match any walked node (orphan image fills), add them as png-fill entries using their nodeName for the filename.
    4. Call `resolveCollisions()` on the full list before returning.

    Filenames: `sanitizeFilename(nodeName) + extension` where extension is `.svg` for SVG and `.png` for png-fill.
