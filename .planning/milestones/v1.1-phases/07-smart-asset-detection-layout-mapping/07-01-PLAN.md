    - detectCompositions({ id: '1', type: 'FRAME', children: [simple 3-vector group] }) => empty Set (no composition)

    - detectCompositions with BOOLEAN_OPERATION node that has a non-normal blendMode child => flagged
    - detectCompositions with standalone BOOLEAN_OPERATION (no visual effects) => NOT flagged
    - Nested composition: outer contains inner => only outer id in Set (outer wins, no recurse)
    - detectCompositions returns warnings array with "Auto-detected {name} as a composition" for each flagged node
    - Structural signals: childCount >= CHILD_COUNT_THRESHOLD (5), maxDepth >= NESTING_DEPTH_THRESHOLD (3), or type is BOOLEAN_OPERATION
    - Visual effect signals: non-normal/non-PASS_THROUGH blendMode, isMask === true, or opacity < 1 on a child (scanned up to SCAN_DEPTH_LIMIT of 3 levels)
    - Both structural AND visual required for flagging
  </behavior>

  <name>Task 2: Breadcrumb path builder</name>
  <files>src/assets/breadcrumb.ts, src/assets/breadcrumb.test.ts</files>
  <behavior>
    - buildBreadcrumbMap([root with named children]) => Map with nodeId -> "Root > Child" paths
    - buildBreadcrumbMap skips generic names matching /^(Frame|Group|Rectangle|Ellipse|Vector|Section|Instance|Line|Star|Polygon)\s*\d*$/i
    - "Frame 427" is generic (skipped), "Frame Header" is NOT generic (kept)
    - "Group" (bare, no number) is generic (skipped)
    - Path with <= 4 segments: "A > B > C > D" (full)
    - Path with 5+ segments: "A > ... > D > E" (smart truncation: first + ... + last 2)
    - Root node itself gets breadcrumb "" (empty) if generic, or its own name if named
    - Deeply nested node under generic parents inherits only the named ancestors in its path
    - Empty tree returns empty Map
  </behavior>
  <action>
    **Step 1: Create breadcrumb.ts** as a new file `src/assets/breadcrumb.ts`:
    - Export `GENERIC_NAME_PATTERN = /^(Frame|Group|Rectangle|Ellipse|Vector|Section|Instance|Line|Star|Polygon)\s*\d*$/i`
    - Export `buildBreadcrumbMap(rootNodes: LayoutNode[]): Map<string, string>`
    - Internal helpers: `walkForBreadcrumbs(node, ancestorPath, map)`, `formatBreadcrumb(parts)`, `isGenericName(name)`
    - `isGenericName`: test name against GENERIC_NAME_PATTERN
    - `walkForBreadcrumbs`: for each node, record `map.set(node.id, formatBreadcrumb(currentPath))`. For children: if child name is generic, pass same path; if named, pass [...path, child.name]. Recurse.
    - `formatBreadcrumb`: 0 parts -> "", 1-4 parts -> parts.join(' > '), 5+ parts -> `${parts[0]} > ... > ${parts[parts.length - 2]} > ${parts[parts.length - 1]}`
    - `buildBreadcrumbMap`: for each root, compute initial path (empty if generic, [root.name] if named), then walkForBreadcrumbs

    **Step 2: Write tests** in `src/assets/breadcrumb.test.ts`:
    Use same `node()` helper pattern. Cover: named paths, generic name skipping, smart truncation at 5+ levels, mixed generic/named paths, empty tree, "Frame Header" kept vs "Frame 427" skipped, bare "Group" skipped.
