## Tech Debt (4 items across 2 phases)

### Phase 1: Plugin Foundation & Figma Connection

1. **usePluginContext returns null** — Plan specified `usePluginContext(): PluginContextValue` but implementation returns `PluginContextValue | null`. All consumers guard with `ctx?.shell ?? null`. Safer but diverges from plan type signature. No functional impact.

2. **plugin.json missing `entry` field** — Original plan specified `"entry": "dist/index.js"`. Current `plugin.json` omits it. If Ship Studio discovers plugins by convention (looking for `dist/index.js`), this is benign. If the field is required, the plugin won't load. Human verification during Phase 1 passed with the current file, suggesting it's optional.

### Phase 5: Brief Assembly & Output

3. **pbcopy is macOS-only** — `copyToClipboard()` in `src/brief/io.ts` pipes through `pbcopy`. On Linux/Windows hosts, BREF-05 will fail with an error toast (graceful degradation, no crash). No fix needed if Ship Studio is macOS-only; otherwise needs a cross-platform clipboard solution.

4. **Silent failure when ctx.project.path unavailable** — `runAssetExport()` in `MainView.tsx` silently returns when `ctx.project.path` is null. User sees extraction succeed but asset export and brief generation never start, with no error message. Recommend adding an error toast: `"Project path unavailable. Cannot export assets."` Affects PLUI-04 (clear error messages) in this edge case.
