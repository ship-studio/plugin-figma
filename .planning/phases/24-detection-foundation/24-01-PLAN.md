<feature>
  <name>@S- prefix asset detection</name>
  <files>src/assets/types.ts, src/assets/detect.ts, src/assets/detect.test.ts</files>
  <behavior>
    DETECT-01 (prefix matching):
    - Tree with children `@S-hero`, `Regular Layer` -> detects only `@S-hero`
    - Case-insensitive: `@s-icon`, `@S-Icon`, `@S-ICON` all detected
    - Strict dash: `@Shero`, `@S hero` NOT detected
    - Empty after prefix: `@S-` alone skipped with warning
    - Root node with @S- prefix is detected

    DETECT-02 (PNG format):
    - `@S-hero` containing child RECTANGLE with IMAGE fill -> format `png`, filename `hero.png`
    - `@S-card` containing INSTANCE whose child has IMAGE fill -> format `png` (walk enters instances)
    - Direct IMAGE fill on the @S- node itself -> format `png`
    - Invisible IMAGE fill (fill.visible===false) does NOT trigger PNG

    DETECT-03 (SVG format):
    - `@S-icon` with only VECTOR children -> format `svg`, filename `icon.svg`
    - `@S-tagline` with only TEXT children -> format `svg` (SVG is default, TEXT-only is safe per svg_outline_text=true)
    - Mixed vector + text (no IMAGE) -> format `svg`

    DETECT-04 (filename stripping):
    - `@S-hero-image` -> `hero-image.png` (prefix stripped BEFORE sanitize)
    - `@S-HeroImage` -> `hero-image.png` (lowercased via sanitize)
    - Three instances each with `@S-icon` -> single `icon.svg` (dedup, not 3 copies)
    - Different original names producing same sanitized name -> `icon.svg`, `icon-2.svg` (collision resolution)

    DETECT-05 (position metadata):
    - Detected asset at depth 3 inside "Page > Hero > Content" -> depth=3, parentPath=["Page", "Hero", "Content"]
    - Each asset carries its nodeId for downstream layout tree cross-referencing

    Nesting rules:
    - `@S-hero` contains `@S-icon` -> only `@S-hero` detected (outermost wins)
    - `@S-icon` inside non-@S- INSTANCE -> detected (walk enters all subtrees)
    - Hidden layer (visible===false) with @S- -> skipped entirely, children not walked
  </behavior>
  <implementation>
    1. Add DetectedAsset and DetectionResult types to src/assets/types.ts (new peer types, do NOT extend ManualAsset or AssetEntry)
    2. Create src/assets/detect.ts with:
       - ASSET_PREFIX_REGEX = /^@s-/i
       - isAssetMarker(name) -> boolean
       - stripAssetPrefix(name) -> string
       - subtreeHasImageFill(node) -> boolean (recursive, enters INSTANCE children, skips invisible fills)
       - walkForAssets(node, depth, parentPath, insideDetected, results, warnings) -> void (recursive walker)
       - deduplicateAndResolve(raws) -> DetectedAsset[] (dedup by filename, then collision resolution)
       - detectAssets(rootNode) -> DetectionResult (public entry point)
    3. Strip @S- prefix BEFORE calling sanitizeFilename() to avoid `s-hero` output (STATE.md decision)
    4. Use `node.visible === false` (strict equality) for hidden check -- undefined means visible
    5. Text-only layers default to SVG (research recommendation: svg_outline_text=true makes this safe)
  </implementation>

    1. **Imports:**
    ```typescript
    import { sanitizeFilename } from './sanitize';
    import { resolveFilenameCollision } from './resolve';
    import type { DetectedAsset, DetectionResult } from './types';
    ```

    2. **Constants and helpers:**
    - `ASSET_PREFIX_REGEX = /^@s-/i`
    - `isAssetMarker(name: string): boolean` -- test prefix regex
    - `stripAssetPrefix(name: string): string` -- replace prefix regex with empty string
    - `subtreeHasImageFill(node: any): boolean` -- recursive, checks `node.fills` for `type === 'IMAGE'` with `fill.visible !== false`, enters all children including INSTANCE subtrees. CRITICAL: check `fill.visible !== false` (not `fill.visible === true`) because visible defaults to true when omitted.

    3. **Internal RawDetection type:**
    ```typescript
    interface RawDetection {
      nodeId: string;
      nodeName: string;
      nameAfterPrefix: string;
      format: 'png' | 'svg';
      depth: number;
      parentPath: string[];
    }
    ```

    4. **walkForAssets(node, depth, parentPath, insideDetected, results, warnings):**
    - If `node.visible === false` -> return immediately (skip hidden + all children)
    - Check `isAssetMarker(node.name)` and `!insideDetected`:
      - Strip prefix, check empty -> push warning and skip if empty
      - Call `subtreeHasImageFill(node)` -> format 'png' or 'svg'
      - Push RawDetection with nodeId, nodeName, nameAfterPrefix, format, depth, parentPath (copy)
      - Set `insideDetected = true` for child recursion
    - Recurse into `node.children` (if array), passing `[...parentPath, node.name]`, depth+1

    5. **deduplicateAndResolve(raws: RawDetection[]): DetectedAsset[]:**
    - Phase 1: Group by `sanitizeFilename(raw.nameAfterPrefix) + '.' + raw.format` -> keep first, drop dupes (silent, per Phase 9 pattern)
    - Phase 2: Run `resolveFilenameCollision` on the deduplicated list for any remaining collisions from different original names
    - Return DetectedAsset array with filename, format, nodeId, nodeName, depth, parentPath

    6. **detectAssets(rootNode: any): DetectionResult:**
    ```typescript
    const raws: RawDetection[] = [];
    const warnings: string[] = [];
    walkForAssets(rootNode, 0, [], false, raws, warnings);
    const assets = deduplicateAndResolve(raws);
    return { assets, warnings };
    ```
