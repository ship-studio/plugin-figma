    NORMALIZATION (normalize.ts + types.ts):
    - Test: Node with layoutGrow: 1 -> result.layoutGrow === 1
    - Test: Node with layoutGrow: 0 -> result.layoutGrow is undefined (don't store default)
    - Test: Node without layoutGrow property -> result.layoutGrow is undefined
    - Test: Node with layoutAlign: 'STRETCH' -> result.layoutAlign === 'STRETCH'
    - Test: Node with layoutAlign: 'INHERIT' -> result.layoutAlign is undefined (don't store default)
    - Test: Node with layoutAlign: 'MIN' -> result.layoutAlign is undefined (not actionable)
    - Test: Node without layoutAlign property -> result.layoutAlign is undefined
    - Test: Absolute child at (150, 80) inside parent at (100, 50) -> absoluteOffset = { top: 30, left: 50 }
    - Test: Absolute child with fractional coords -> absoluteOffset values are Math.round'd
    - Test: Absolute child but parent has no bbox (null) -> no absoluteOffset (no crash)
    - Test: Absolute child but child has no absoluteBoundingBox (hidden) -> no absoluteOffset
    - Test: Non-absolute child (positioning AUTO) -> no absoluteOffset even when parent bbox available
    - Test: normalizeNode parentBBox parameter is optional, defaults to undefined (backward compat)
    - Test: Recursive children receive parent's absoluteBoundingBox as parentBBox
    - Test: normalizeTree passes null as parentBBox for root nodes (no parent)

    BRIEF RENDERING (generate.ts):
    - Test: Node with positioning ABSOLUTE + absoluteOffset {top:16, left:24} -> output contains "[absolute] top:16 left:24"
    - Test: Node with positioning ABSOLUTE but no absoluteOffset -> output contains "[absolute]" (existing behavior preserved)
    - Test: Node with layoutGrow: 1 -> inline styles contain "flex-grow:1"
    - Test: Node with layoutAlign: 'STRETCH' -> inline styles contain "align-self:stretch"
    - Test: Node with both layoutGrow: 1 AND layoutAlign: 'STRETCH' -> both present in inline styles
    - Test: Node with no flex-child properties -> no flex-grow or align-self in output

    Expected brief output format:
    ```
    Frame 'Hero Section' (column, gap: 24, padding: 32) 1200x800
      Frame 'Badge' 50x50 [absolute] top:16 left:16
      Text 'Title' (Inter 48/700) 800x60 {flex-grow:1}
      Frame 'Content Row' (row, gap: 16) 800x400 {align-self:stretch}
    ```

  <implementation>
    1. Add 3 optional fields to LayoutNode interface in types.ts:
       - layoutGrow?: 0 | 1
       - layoutAlign?: 'INHERIT' | 'STRETCH'
       - absoluteOffset?: { top: number; left: number }

    2. In normalize.ts, add optional parentBBox parameter to normalizeNode:
       - Signature: normalizeNode(figmaNode, components, depth, parentBBox?: { x: number; y: number } | null)
       - After the existing layoutPositioning block (~line 107), add:
         - Read layoutGrow: only store when === 1 (skip 0, the default)
         - Read layoutAlign: only store when === 'STRETCH' (skip INHERIT/MIN/CENTER/MAX)
       - After positioning capture, compute absoluteOffset:
         - Guard: result.positioning === 'ABSOLUTE' && parentBBox && node.absoluteBoundingBox
         - Compute: { top: Math.round(child.y - parent.y), left: Math.round(child.x - parent.x) }
       - In the children recursion block (~line 181-187):
         - Extract myBBox from node.absoluteBoundingBox if present, else null
         - Pass myBBox as 4th arg to recursive normalizeNode calls
       - In normalizeTree, pass null as parentBBox for root nodes

    3. In generate.ts renderNodeLine:
       - Modify the positioning block (line 269-271): when absoluteOffset exists, render "top:N left:N" after [absolute]
       - In buildInlineStyles (line 303): add flex-grow:1 when layoutGrow === 1, add align-self:stretch when layoutAlign === 'STRETCH'

    Anti-patterns (from research):
    - Do NOT use relativeTransform for offset calculation
    - Do NOT subtract parent padding from offset (raw subtraction is correct for CSS position:absolute)
    - Do NOT render layoutAlign: INHERIT or MIN/CENTER/MAX (noise; parent's alignItems covers those)
    - Do NOT render layoutGrow: 0 (fixed-size default, noise)
  </implementation>
