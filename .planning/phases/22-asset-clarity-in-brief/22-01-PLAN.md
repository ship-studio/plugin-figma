<task type="auto">
  <name>Task 1: Add Usage column to Assets table and derive usage context</name>
  <files>src/brief/generate.ts</files>
  <action>
  Modify `buildAssetsSection` in `src/brief/generate.ts` to add a Usage column that describes each asset's intended purpose.

  **Table header change:**
  Change from `| File | Type | Location | Path |` to `| File | Type | Usage | Path |`

  **Remove the Location column** and replace it with a new Usage column that combines the asset type with its breadcrumb location into a human-readable purpose string:

  **Create a new `deriveUsageContext` function** (module-private) that takes assetType and breadcrumb location, returning a usage description:
  - If assetType is `'icon'` and location is non-empty (not `'--'`): return `"Icon in {location}"` (e.g., "Icon in Hero > Header")
  - If assetType is `'icon'` and location is `'--'`: return `"Icon"`
  - If assetType is `'image'` and location is non-empty: return `"Image in {location}"` (e.g., "Image in Hero Section > Card")
  - If assetType is `'image'` and location is `'--'`: return `"Image"`
  - For the preview row: use `"Full-page preview screenshot"` as the usage
  - Default (no assetType, e.g. `'File'`): return `"Asset in {location}"` or `"Asset"` if no location

  **Update the preview row:**
  Change from `| preview.png | Preview | -- | {path} |` to `| preview.png | Preview | Full-page preview screenshot | {path} |`

  **Update each asset row** to use `deriveUsageContext(asset.assetType, location)` instead of the raw breadcrumb location.

  **Also update `buildInstructionsSection`:** In the shared `sharedDuring` string, change the sentence to be more explicit about the Assets section being the definitive manifest. Update from:
  `"Use only the assets listed in the Assets section below. If an asset is missing, ask the user rather than substituting or fabricating a replacement."`
  to:
  `"The Assets section below is the complete manifest of provided files. Use only these assets -- every visual element NOT listed there should be built with CSS or HTML, not with image files. If you need an asset that is not listed, ask the user rather than substituting or fabricating a replacement."`

  This is purely a brief generator change -- no UI or pipeline changes needed.
