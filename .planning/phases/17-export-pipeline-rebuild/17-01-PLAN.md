<feature>
  <name>Export pipeline with ManualAsset[] and format-aware batching</name>
  <files>src/assets/export.ts, src/assets/export.test.ts</files>
  <behavior>
    ## Core Behaviors

    1. **Format partitioning:** Given ManualAsset[] with mixed formats, partition into PNG and SVG groups. Call fetchImages once per non-empty partition. PNG uses scale=2. SVG omits scale.
       - Input: [{nodeId:"1:1", format:"png", status:"valid"}, {nodeId:"2:2", format:"svg", status:"valid"}]
       - Expected: Two fetchImages calls — one with format='png',scale=2 for ["1:1"], one with format='svg' for ["2:2"]

    2. **Only valid assets exported:** Only ManualAssets with status='valid' are included in the export. Entries with status='error' or 'resolving' are silently skipped.
       - Input: [{status:"valid", nodeId:"1:1"}, {status:"error", nodeId:"2:2"}]
       - Expected: Only nodeId "1:1" is passed to fetchImages

    3. **Null URL warning:** When fetchImages returns null for a nodeId, push a warning string and skip that asset's download. Do not throw.
       - Input: asset with nodeId "3:3", fetchImages returns {"3:3": null}
       - Expected: warnings includes "Failed to render {filename}: Figma returned no image for node 3:3", asset NOT in result.assets

    4. **Download failure warning:** When downloadFile returns {success:false}, push a warning and continue. Do not throw.
       - Input: asset with valid URL, downloadFile returns {success:false, error:"timeout"}
       - Expected: warnings includes "Failed to download {filename}: timeout", asset NOT in result.assets

    5. **Successful download result shape:** Each downloaded asset appears in result.assets with correct fields.
       - Input: ManualAsset{nodeId:"1:1", filename:"hero.png", format:"png"}
       - Expected: result.assets includes {filename:"hero.png", path:"{assetsDir}/hero.png", nodeId:"1:1", assetType:"image"}
       - SVG variant: format="svg" -> assetType="icon"

    6. **Preview always generated:** Preview PNG is fetched and downloaded regardless of manual assets. Preview uses selectedNodeId at 2x scale.
       - Input: empty manualAssets
       - Expected: previewPath is set, fetchImages called once for preview

    7. **Progress callbacks:** onProgress fires with phase='preview' at current=0, then phase='assets' for each manual asset at current=1..N. Total = manualAssets.length + 1.

    8. **Empty asset list:** When manualAssets is empty or omitted, pipeline produces preview only with assets=[].

    9. **Node ID encoding fallback:** When looking up a nodeId in the fetchImages response map, try the raw ID first, then encodeURIComponent(nodeId), then decodeURIComponent(nodeId). This handles potential encoding mismatches from the Figma API.

    ## Edge Cases

    10. **All assets fail to render:** Every fetchImages URL is null. Result has empty assets[], all warnings collected, no throw.
    11. **fetchImages throws:** If the fetchImages call itself throws (network error), catch it, push a warning for the entire batch, and continue with other format partition.
    12. **Preview fails but assets succeed:** Preview warning is added but assets still export normally.
  </behavior>
