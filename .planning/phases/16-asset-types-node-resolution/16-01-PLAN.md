    ## I-Prefix Detection (AINP-05)
    - isInstanceChildId('I20:1;20:2') -> true
    - isInstanceChildId('I1:2;3:4') -> true
    - isInstanceChildId('12:34') -> false
    - isInstanceChildId('0:1') -> false
    - isInstanceChildId('') -> false
    - extractParentInstanceId('I20:1;20:2') -> '20:1'
    - extractParentInstanceId('I1:2;3:4') -> '1:2'
    - extractParentInstanceId('12:34') -> null
    - extractParentInstanceId('Ifoo') -> null (no semicolon)

    ## Format Suggestion (AINP-06)
    - suggestFormat('VECTOR') -> 'svg'
    - suggestFormat('LINE') -> 'svg'
    - suggestFormat('STAR') -> 'svg'
    - suggestFormat('ELLIPSE') -> 'svg'
    - suggestFormat('REGULAR_POLYGON') -> 'svg'
    - suggestFormat('BOOLEAN_OPERATION') -> 'svg'
    - suggestFormat('FRAME') -> 'png'
    - suggestFormat('GROUP') -> 'png'
    - suggestFormat('COMPONENT') -> 'png'
    - suggestFormat('INSTANCE') -> 'png'
    - suggestFormat('TEXT') -> 'png'
    - suggestFormat('RECTANGLE') -> 'png'
    - suggestFormat('UNKNOWN_TYPE') -> 'png' (fallback)

    ## Filename Collision Resolution (NAME-02)
    - resolveFilenameCollision('icon.png', []) -> 'icon.png'
    - resolveFilenameCollision('icon.png', ['icon.png']) -> 'icon-2.png'
    - resolveFilenameCollision('icon.png', ['icon.png', 'icon-2.png']) -> 'icon-3.png'
    - resolveFilenameCollision('icon.svg', ['icon.png']) -> 'icon.svg' (different extension, no collision)
    - resolveFilenameCollision('logo', []) -> 'logo' (no extension edge case)

    ## Asset Derivation (NAME-01, NAME-02, AINP-06)
    - deriveAssetFromNode('12:34', 'Hero Image', 'FRAME', []) -> { nodeId: '12:34', nodeName: 'Hero Image', filename: 'hero-image.png', format: 'png', status: 'valid', warning: undefined }
    - deriveAssetFromNode('5:6', 'Arrow', 'VECTOR', []) -> { ..., filename: 'arrow.svg', format: 'svg', status: 'valid' }
    - deriveAssetFromNode('7:8', 'Frame 427', 'FRAME', []) -> { ..., filename: 'frame-427.png', warning: "Auto-named: frame-427.png -- consider renaming" }
    - deriveAssetFromNode('9:10', 'Icon', 'VECTOR', [{ filename: 'icon.svg', ... }]) -> { ..., filename: 'icon-2.svg' }

    ### Step 1: Add ManualAsset type to src/assets/types.ts

    Add the ManualAsset interface alongside the existing AssetEntry (which is kept for Phase 17/18 transition):

    ```typescript
    export interface ManualAsset {
      /** Figma node ID (e.g., "12:34") */
      nodeId: string;
      /** Original Figma layer name, for display */
      nodeName: string;
      /** Sanitized filename WITH extension (e.g., "hero-image.png") */
      filename: string;
      /** User-selected or auto-suggested format */
      format: 'png' | 'svg';
      /** Lifecycle status */
      status: 'resolving' | 'valid' | 'error';
      /** Error message when status is 'error' */
      error?: string;
      /** Warning message (e.g., "Auto-named: frame-427.png -- consider renaming") */
      warning?: string;
    }
    ```

    ### Step 2: Create src/assets/resolve.ts with all pure functions

    Import existing utilities:
    - `sanitizeFilename` from `./sanitize`
    - `GENERIC_NAME_PATTERN` from `./breadcrumb`
    - `fetchFileNodes` from `../figma-api`
    - `ManualAsset` from `./types`
    - `Shell` from `../types`

    Implement these exported functions:

    **isInstanceChildId(nodeId: string): boolean**
    - Return `nodeId.startsWith('I')` (per CONTEXT.md: client-side I-prefix check)

    **extractParentInstanceId(nodeId: string): string | null**
    - If not I-prefix, return null
    - Strip leading 'I', find semicolon index
    - If no semicolon, return null
    - Return substring before semicolon

    **suggestFormat(nodeType: string): 'png' | 'svg'**
    - Define `VECTOR_NODE_TYPES = new Set(['VECTOR', 'LINE', 'STAR', 'ELLIPSE', 'REGULAR_POLYGON', 'BOOLEAN_OPERATION'])`
    - Return `VECTOR_NODE_TYPES.has(nodeType) ? 'svg' : 'png'`

    **resolveFilenameCollision(candidateFilename: string, existingFilenames: string[]): string**
    - If candidateFilename not in existingFilenames, return it as-is
    - Split at last dot to get base and extension
    - Start counter at 2, increment until `${base}-${counter}${ext}` is not in existingFilenames
    - Return the unique filename
    - Handle no-extension edge case (no dot in filename)

    **deriveAssetFromNode(nodeId: string, nodeName: string, nodeType: string, existingAssets: ManualAsset[]): ManualAsset**
    - Call suggestFormat(nodeType) for format
    - Call sanitizeFilename(nodeName) for base name
    - Build candidate filename: `${baseName}.${format}`
    - Extract existingFilenames from existingAssets.map(a => a.filename)
    - Call resolveFilenameCollision(candidate, existingFilenames) for final filename
    - Check GENERIC_NAME_PATTERN.test(nodeName) for warning -- use the FINAL filename (after collision resolution, per Pitfall 5 in research)
    - Warning text: `Auto-named: ${finalFilename} -- consider renaming`
    - Return ManualAsset with status 'valid'

    **resolveNode(shell: Shell, token: string, fileKey: string, nodeId: string, existingAssets: ManualAsset[]): Promise<ManualAsset>**
    - Call fetchFileNodes(shell, token, fileKey, nodeId)
    - Extract rootNode.name and rootNode.type
    - Call deriveAssetFromNode(nodeId, rootNode.name, rootNode.type, existingAssets)
    - Return the ManualAsset
    - On error: return ManualAsset with status 'error', error message from caught exception, empty nodeName/filename, 'png' format default

    ### Step 3: Create src/assets/resolve.test.ts

    Follow existing test patterns (vitest describe/it/expect). Test all behaviors listed above. For resolveNode, test the error case with a mock shell that rejects.

    **Do NOT mock fetchFileNodes** for the happy-path resolveNode test -- instead test `deriveAssetFromNode` thoroughly (it's the pure core) and add a minimal test for `resolveNode` error handling where the shell throws.

    ### Key rules from CONTEXT.md (locked decisions):
    - Format field: 'png' | 'svg' only, no 'auto' variant
    - Status tracking: 'resolving' | 'valid' | 'error'
    - Store both nodeName (display) and filename (sanitized)
    - Filename is auto-derived on add, mutable for Phase 19 edits
    - Use leaf node name only -- no parent path in filename
    - Collisions resolved at add-time, not export-time
    - I-prefix check is client-side before any API call
    - Soft rejection for I-prefix: don't add, show warning
    - Format pre-selected based on node type, user can override silently
