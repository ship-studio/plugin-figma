<task type="auto">
  <name>Task 1: Add Figma API fetch functions and extraction orchestrator</name>
  <files>src/figma-api.ts, src/layout/extract.ts, src/types.ts</files>
  <action>
    **In src/figma-api.ts**, add two new exported functions (keep all existing exports):

    1. `fetchFileNodes(shell, token, fileKey, nodeId)`:
       - Calls `figmaApiCall<GetFileNodesResponse>(shell, '/files/${fileKey}/nodes?ids=${encodeURIComponent(nodeId)}', token)`
       - Use a longer timeout for this call: `{ timeout: 120000 }` (2 minutes for potentially large subtrees)
       - Returns `{ rootNode: response.nodes[nodeId].document, components: response.nodes[nodeId].components }`
       - Throws descriptive error if nodeId not found in response

    2. `fetchFullFile(shell, token, fileKey)`:
       - Calls `figmaApiCall<GetFileResponse>(shell, '/files/${fileKey}', token)`
       - Use longer timeout: `{ timeout: 120000 }`
       - Returns `{ rootNodes: response.document.children, components: response.components }`
       - Note: `response.document.children` are CanvasNode[] (pages). For page scope, we want first page's children, but the extract orchestrator will handle page selection.

    Import `GetFileResponse` and `GetFileNodesResponse` from `@figma/rest-api-spec`.

    **In src/layout/extract.ts**, create the extraction orchestrator:

    ```typescript
    export interface ExtractLayoutOptions {
      shell: Shell;
      token: string;
      fileKey: string;
      nodeId: string | null;
      scope: ExtractionScope;
    }

    export interface ExtractLayoutResult {
      extraction: ExtractionResult;
      // Large tree warning: set when nodeCount exceeds threshold
      largeTreeWarning?: { nodeCount: number; message: string };
    }
    ```

    Implement `extractLayout(options: ExtractLayoutOptions): Promise<ExtractLayoutResult>`:

    1. **Fetch based on scope:**
       - `scope === 'node'` or `scope === 'frame'`: Call `fetchFileNodes(shell, token, fileKey, nodeId!)`. Throws if nodeId is null.
       - `scope === 'page'`: Call `fetchFullFile(shell, token, fileKey)`. Return first page's children as root nodes (pages are CanvasNode[] -- use `rootNodes[0]?.children || []` since each page is a canvas with child frames).

    2. **Count nodes before normalization:**
       - Sum `countNodes` across all root nodes from the API response
       - If count > WARN_THRESHOLD (500): set `largeTreeWarning` with count and message "This selection has ~{count} nodes. Large extractions may produce verbose output."
       - If count > MAX_THRESHOLD (2000): still proceed but set `truncated: true` in result

    3. **Normalize:**
       - Call `normalizeTree(rootNodes, components)` from 02-01
       - If nodeCount exceeded MAX_THRESHOLD, set extraction.truncated = true

    4. **Return** `{ extraction, largeTreeWarning? }`

    Export WARN_THRESHOLD (500) and MAX_THRESHOLD (2000) as named constants.

    **In src/types.ts**, no changes needed -- ExtractionScope is already defined. The new ExtractionResult type lives in src/layout/types.ts.
  </action>
