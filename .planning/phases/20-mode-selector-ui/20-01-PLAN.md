**3. Render the mode selector:**

Insert the mode selector JSX between the "Scope hint" section and the "Asset List Panel" section. It uses the same visibility gate as both: `parsedUrl && fileInfo && !validating`.

Additionally, hide the mode selector when results are showing (when `briefResult` is truthy). This matches CONTEXT.md: "Hidden when results card is showing. Reappears when user clicks 'Get New Brief'."

The JSX to insert (after the scope hint closing `</div>` around line 519, before the AssetListPanel):

```tsx
{/* Brief Mode Selector -- visible after URL validation, hidden during results */}
{parsedUrl && fileInfo && !validating && !briefResult && (
  <div className="figma-plugin-mode-section">
    <span className="figma-plugin-mode-label">Brief mode</span>
    <div className="figma-plugin-mode-group">
      {BRIEF_MODES.map((mode) => (
        <div
          key={mode.id}
          className={`figma-plugin-mode-card${briefMode === mode.id ? ' selected' : ''}`}
          onClick={() => setBriefMode(mode.id)}
          role="radio"
          aria-checked={briefMode === mode.id}
          tabIndex={0}
          onKeyDown={(e) => {
            if (e.key === 'Enter' || e.key === ' ') {
              e.preventDefault();
              setBriefMode(mode.id);
            }
          }}
        >
          <div className="figma-plugin-mode-card-name">{mode.name}</div>
          <div className="figma-plugin-mode-card-desc">{mode.description}</div>
        </div>
      ))}
    </div>
  </div>
)}
```

Key implementation notes:
- `!briefResult` in the condition hides the selector when results card is showing. When the user clicks "Get New Brief", `briefResult` gets cleared (line ~368: `setBriefResult(null)`), making the mode selector reappear automatically.
- `role="radio"` and `aria-checked` for accessibility.
- `tabIndex={0}` and `onKeyDown` for keyboard navigation.
- Do NOT add the mode to the `handleUrlChange` reset logic or any other reset -- it persists across all operations within the session.
