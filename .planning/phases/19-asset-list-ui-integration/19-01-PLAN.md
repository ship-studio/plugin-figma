Create `src/components/AssetListPanel.tsx` as a controlled React component that renders the complete asset management UI.

**Props interface:**

```typescript
import { useState, useCallback, useRef } from 'react';
import type { ChangeEvent, KeyboardEvent } from 'react';
import { parseFigmaUrl } from '../url-parser';
import { isInstanceChildId, resolveNode, resolveFilenameCollision } from '../assets/resolve';
import { sanitizeFilename } from '../assets/sanitize';
import type { ManualAsset } from '../assets/types';
import type { Shell } from '../types';

interface AssetListPanelProps {
  /** File key from the main design URL -- for same-file validation (AINP-03) */
  designFileKey: string | null;
  /** Current list of manual assets */
  assets: ManualAsset[];
  /** Callback when user adds an asset (optimistic placeholder) */
  onAdd: (asset: ManualAsset) => void;
  /** Callback when user removes an asset by nodeId */
  onRemove: (nodeId: string) => void;
  /** Callback when user clears all assets */
  onClear: () => void;
  /** Callback when user renames an asset */
  onRename: (nodeId: string, newFilename: string) => void;
  /** Callback when async resolution completes */
  onResolved: (nodeId: string, resolved: ManualAsset) => void;
  /** Callback when user toggles format (also updates filename extension) */
  onFormatChange: (nodeId: string, newFormat: 'png' | 'svg', newFilename: string) => void;
  /** Whether the export pipeline is running (disables all interactions) */
  disabled?: boolean;
  /** Shell for API calls */
  shell: Shell;
  /** Figma API token */
  token: string;
}
```

**Component implementation:**

1. **State:** `assetUrlInput` (string), `addError` (string | null), `editingNodeId` (string | null), `editValue` (string).

2. **handleAddAsset** (useCallback, async):
   - Guard: return if `assetUrlInput.trim()` empty, or `designFileKey` is null, or `disabled`.
   - Parse URL: `const parsed = parseFigmaUrl(assetUrlInput)`.
   - Validate (in order, return on first failure with `setAddError`):
     - `!parsed` -> "Invalid Figma URL"
     - `!parsed.nodeId` -> "URL must point to a specific element (include node-id)" (AINP-04)
     - `parsed.fileKey !== designFileKey` -> "Asset must be from the same Figma file" (AINP-03)
     - `assets.some(a => a.nodeId === parsed.nodeId)` -> "This element is already in the list" (LIST-04)
     - `isInstanceChildId(parsed.nodeId)` -> "This is an instance child -- select the parent component instead" (AINP-05 UX reinforcement)
   - Optimistic add: create placeholder ManualAsset with `status: 'resolving'`, empty nodeName/filename, format 'png'. Call `onAdd(placeholder)`.
   - Clear input and error: `setAssetUrlInput('')`, `setAddError(null)`.
   - Async resolve: `const resolved = await resolveNode(shell, token, designFileKey, parsed.nodeId, assets)`. Call `onResolved(parsed.nodeId, resolved)`.
   - Dependency array: `[assetUrlInput, designFileKey, assets, shell, token, disabled, onAdd, onResolved]`.

3. **handleFormatToggle** (useCallback):
   - Takes `(asset: ManualAsset)`.
   - New format: `asset.format === 'png' ? 'svg' : 'png'`.
   - Compute new filename: strip old extension, add new extension, run `resolveFilenameCollision` against other assets' filenames.
   - Call `onFormatChange(asset.nodeId, newFormat, newFilename)`.

4. **startEdit / saveEdit** (inline filename editing):
   - `startEdit(nodeId, currentFilename)`: set `editingNodeId`, strip extension from filename for `editValue`.
   - `saveEdit(nodeId, format)`: sanitize `editValue`, add extension, run `resolveFilenameCollision` against other assets. Call `onRename(nodeId, finalFilename)`. Clear `editingNodeId`.
   - On Enter key in edit input: call saveEdit.
   - On blur in edit input: call saveEdit.

5. **Render structure:**

```

            </span>
          )}
          {asset.status === 'valid' && (
            <span className="figma-plugin-asset-status-valid">&#10003;</span>
          )}
          {asset.status === 'error' && (
            <span className="figma-plugin-asset-status-error" title={asset.error}>&#10007;</span>
          )}

          {/* Format badge (clickable toggle) */}
          {asset.status === 'valid' && (
            <span

            >

            </span>
          )}

          {editingNodeId === asset.nodeId ? (

              value={editValue}

              onBlur={() => saveEdit(asset.nodeId, asset.format)}

              title={asset.status === 'valid' ? 'Click to rename' : asset.error || 'Resolving...'}
            >
              {asset.status === 'resolving' ? 'Resolving...' : asset.status === 'error' ? (asset.error || 'Failed to resolve') : asset.filename}
            </span>
          )}

          {/* Warning */}
          {asset.warning && (
            <span className="figma-plugin-asset-warning" title={asset.warning}>&#9888;</span>
          )}

          {/* Remove button */}
          <button
            className="figma-plugin-asset-remove-btn"
            onClick={() => onRemove(asset.nodeId)}
            disabled={disabled}
            title="Remove asset"
          >
            &times;
          </button>
        </div>
      ))}
    </>
  )}
</div>
```

**Important implementation details:**
- Use `useRef` to track in-flight nodeIds to prevent the duplicate race condition (Pitfall 2 from research). Before adding, check the ref Set. Add nodeId on initiation, remove on resolution.
- Clear `addError` on input change (not just on successful add).
- The component does NOT own the `assets` state -- it receives it via props and communicates changes via callbacks. This is a controlled component pattern matching the existing Modal.tsx convention.
- All interactions (add, remove, clear, rename, format toggle) are disabled when `disabled` prop is true.
