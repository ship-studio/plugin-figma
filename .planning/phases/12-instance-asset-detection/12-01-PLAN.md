<objective>
Add three pure-function capabilities to the asset identification engine: (1) instance IMAGE fill override detection (ASSET-06), (2) instance child IMAGE fill recursion (ASSET-05), and (3) simple rectangle filtering (ASSET-07).

Purpose: These are the core detection heuristics that close the gap between what Figma designs contain and what the plugin exports. Without them, hero images inside component instances are invisible, instance background image overrides produce component screenshots instead of the actual image, and dozens of noise SVGs are exported for CSS-reproducible rectangles.

Output: Updated `identify.ts` with three new capabilities and comprehensive test coverage.
</objective>

<feature>
  <name>Instance asset detection and rectangle filtering in identifyAssets</name>
  <files>src/assets/identify.ts, src/assets/identify.test.ts</files>
  <behavior>
    ASSET-07 — Simple rectangle filtering:
    - isSimpleRectangle returns true when: no visible strokes, no gradient fills, no image fills, no visible effects, and all remaining fills are SOLID or empty
    - isSimpleRectangle returns false when: node has visible strokes (even one)
    - isSimpleRectangle returns false when: node has gradient fill (type starts with GRADIENT_)
    - isSimpleRectangle returns false when: node has IMAGE fill
    - isSimpleRectangle returns false when: node has visible effects (shadows, blurs)
    - RECTANGLE nodes passing isSimpleRectangle are silently skipped (no entry, no warning)
    - RECTANGLE nodes failing isSimpleRectangle are still exported as SVG (existing behavior preserved)

    ASSET-06 — Instance IMAGE fill override:
    - When an INSTANCE node with componentRef has an IMAGE fill, it is exported as png-fill (not png-render)
    - The filename uses the instance node's own name (sanitized), not the component name
    - The imageRef comes from imageFillMap (if matched) or from getImageRefFromFills
    - After exporting as png-fill, walkTree returns immediately (no png-render, no child recursion)
    - Instance dedup still applies: same componentId+variant with IMAGE fill override only exported once

    ASSET-05 — Instance child IMAGE fill detection:
    - When an INSTANCE node does NOT have its own IMAGE fill, walkTree recurses into children to find IMAGE fills
    - findImageFillsInChildren walks children at full depth (no limit)
    - Only IMAGE fills are collected from instance children — no SVGs, no rectangles
    - Each child image uses the child node's own name for the filename
    - imageRef dedup is global: same imageRef across multiple instances is exported only once (via seenImageRefs set passed into walkTree)
    - When child images are found, the instance does NOT also get a png-render export
    - When no child images are found AND instance is not deduplicated, it still gets a png-render (existing behavior)
    - Instance component dedup continues to work — when a deduplicated instance is skipped, its child images are still scanned (imageRef dedup handles actual file dedup)
  </behavior>
  <implementation>
    1. Add isSimpleRectangle(node: LayoutNode): boolean function.
       - Check node.strokes for any visible entries (s.visible !== false)
       - Check node.fills for any gradient (type starts with GRADIENT_)
       - Check node.fills via hasImageFill()
       - Check node.effects for any visible entries (e.visible !== false)
       - Return true only if none of the above

    2. Add findImageFillsInChildren(node: LayoutNode, imageFillMap: Map, matchedNodeIds: Set, seenImageRefs: Set): AssetEntry[]
       - Recursive walk of node.children
       - For each child: if hasImageFill, extract imageRef (from imageFillMap or getImageRefFromFills)
       - Dedup by imageRef via seenImageRefs (global set)
       - Push matching entries as png-fill with child's name
       - Continue recursing into child's children
       - Return collected AssetEntry[]

    3. Modify walkTree signature to accept seenImageRefs: Set<string> parameter.

    4. Modify walkTree INSTANCE branch (line 76-88):
       - FIRST: check hasImageFill(node). If true:
         - Get imageRef from imageFillMap or getImageRefFromFills
         - Check seenImageRefs for dedup — if already seen, return
         - Add to seenImageRefs
         - Push png-fill entry with node.name as filename
         - Return (no png-render, no child recursion)
       - SECOND: call findImageFillsInChildren to get child images
       - Instance dedup: check seenInstances as before
         - If new instance AND no child images: push png-render (existing behavior)
         - If new instance AND has child images: do NOT push png-render
       - Push all child image entries

    5. Modify walkTree RECTANGLE branch (line 124-137):
       - Add isSimpleRectangle guard at the top
       - If simple: return immediately (silent skip)
       - Otherwise: existing SVG export logic

    6. Update identifyAssets to create seenImageRefs set and pass to walkTree.
  </implementation>
