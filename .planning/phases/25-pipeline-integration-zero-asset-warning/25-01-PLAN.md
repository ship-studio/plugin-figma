<objective>
Wire the detectAssets() output from Phase 24 into the extraction/export pipeline and add a blocking zero-asset warning when no @S- prefixed layers are found.

Purpose: Detection replaces manual asset URLs as the primary asset source. The zero-asset warning guides designers to use the @S- naming convention, with "Try again" re-fetching from the API so they can fix their Figma file and retry without restarting.

Output: Modified pipeline that detects @S- assets automatically, merges them with manual assets for export, and shows a blocking warning card when zero @S- assets are found.
</objective>
